Halo reads your real Ghostty config and adds a handful of halo- keys on top. Below: those keys with their defaults, the common Ghostty options you inherit for free, the keybindings, and the agent-control CLI.

Halo config keys

Put these in your Ghostty config (or Halo's own config once you import it). libghostty ignores unknown keys, so the halo- keys pass straight through. Every value defaults to Halo's current look — an empty config changes nothing.

KeyDefaultWhat it does
halo-projects — Comma-separated directories shown as projects in the sidebar, e.g. ~/a, ~/b.
halo-accent theme accent Selection / focus / tab accent color, e.g. #889b94. Falls back to the theme accent.
halo-surface ghostty background Window + pane background override, e.g. #161719. Defaults to your Ghostty background.
halo-sidebar-width 224 Sidebar width in pixels.
halo-font-family GeistMono Family for UI / directory / tab / project text.
halo-font-mono MartianMono Family for the uppercase instrument labels.
halo-font-size 13 Base UI font size in points; scales all chrome text.
halo-divider-width 8 Grab/draw thickness of the split divider, in pixels.
Two config locations. Halo reads your live Ghostty config by default (so colors stay in sync). Once you use Import ghostty config or the Settings panel, Halo writes its own file at ~/.config/halo/config (or $XDG_CONFIG_HOME/halo/config) and that one wins. Reload live with halo reload.

Inherited Ghostty keys

Because Halo embeds real libghostty and parses your Ghostty config, any Ghostty key works. Colors (background, foreground, palette) sync straight into Halo. A curated set of common ones:

themeNamed Ghostty theme; resolves color palette for the whole window.
font-familyTerminal (cell) font family.
font-sizeTerminal font size in points.
background / foregroundTerminal background / text colors. Halo's surface defaults to background.
palette = N=#rrggbbOne of the 16 ANSI colors; synced into Halo.
cursor-colorCursor color.
selection-background / selection-foregroundSelection colors.
window-padding-x / window-padding-yInner padding around the terminal grid.
cursor-styleblock, bar, or underline.
background-opacityWindow background opacity (0–1).

Full reference: ghostty.org/docs/config ↗.

Projects & sidebar

List directories in halo-projects to populate the sidebar. Each project holds its own sessions; sessions can be plain shells or git-worktree-isolated branches (see worktree). Toggle the sidebar with ⌘ B.

Keybindings

Halo's built-in shortcuts. Panes live inside sessions; sessions live inside projects.

Panes & sessions

⌘ ⇧ DSplit the focused pane horizontally

⌘ ⇧ WClose the whole session

Agent control CLI

Drive & observe a running Halo from scripts or AI agents. The halo binary talks to the app over a unix socket: launch with no args to start the GUI, or pass a verb to control the running app.

# launch the GUI
$ halo

# drive the running app
$ halo <verb> [args]

$ halo help   # discoverable capability list

Verbs

VerbWhat it does
split [-h|--horizontal] [--cwd DIR]Split the focused pane (vertical by default)
new-pane [--cwd DIR]Open a new pane next to the focused one
closeClose the focused pane
focus [ID]Focus pane ID, or cycle to the next
zoomToggle zoom on the focused pane
send-keys <ID|focused> <text>Type text into a pane
capture [ID|focused] [--scrollback]Print a pane's text — read output back into your agent loop
listList panes / tabs as JSON
open [PATH]Open PATH in a new tab (default ~)
tab new|next|prev|close [--cwd DIR]Manage tabs
worktree <branch> [--base <ref>]Open a git-worktree-isolated session on <branch>
browser [url|port]Open an embedded browser pane (a port becomes http://localhost:PORT)
reloadRe-read the config and apply colors / font / theme live

A closed-loop example

The point of capture is that an agent can read what it did — act, observe, decide:

$ halo worktree fix/login --base main
$ halo split -h
$ halo send-keys focused "npm test"
$ halo capture focused --scrollback   # → scrollback back to the agent

The socket

The control socket lives at:

~/Library/Application Support/halo/control.sock

It's owner-only (0600) — it can inject keystrokes and read scrollback, so no other local user can connect. If the app isn't running, control verbs exit with halo: app not running.
