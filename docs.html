Docs

Halo reads your real Ghostty config, so any Ghostty option works — plus a handful of halo- keys for the chrome. Drive it from scripts and agents over a local control socket.

Halo config keys

Put these in your Ghostty config (or Halo's own config after Settings ▸ Import). libghostty ignores the halo- prefix, so they pass straight through.

KeyDefaultWhat it does
halo-projects—Comma-separated project paths for the sidebar (~/a, ~/b).
halo-accenttheme accentSelection, focus ring, and active-session color.
halo-surfaceghostty backgroundChrome surface — sidebar, titlebar, gaps. For the terminal background itself, set the ghostty background key (Settings ▸ Terminal bg). Set both to the same value for a seamless window.
halo-sidebar-width224Sidebar width, in points.
halo-font-familyGeistMonoSidebar / UI text family.
halo-font-monoMartianMonoInstrument-label family (uppercase chips).
halo-font-size13Base UI font size.
halo-divider-width8Split-divider grab width.
halo-prefixctrl+btmux-style prefix key. Empty value disables prefix mode.
halo-prefix-bind—Rebind a prefix key: <key>:<action>, comma-separated for multiple, e.g. v:split-vertical. Actions: split-vertical, split-horizontal, focus-left/down/up/right, zoom, new-session, next-session, prev-session, rename, detach, kill.
halo-persistonRun sessions through the background daemon so shells survive Halo quitting. Set false for plain in-process shells (no persistence).

Inherited Ghostty keys

Anything in the Ghostty config reference works in Halo. Colors, fonts, and themes sync automatically. A few common ones:

Change anything, then Settings ▸ Reload (or halo reload) — no relaunch.

Sessions & persistence

Sessions run under a small background daemon (halod). Shells survive Halo quitting or crashing — and survive closing a window — then reattach on next launch with native scrollback restored. Long scrollback spills to disk, so history is recovered even if the daemon restarts; inline images replay across reattach.

The sidebar is the single source of truth: every session lives there. Closing a window just closes a view — the sessions stay alive and reopen with the window. Closing a session (its last pane) kills its shell, so there are no hidden orphans.

Open more than one window with ⌘N; they all share the same sidebar and session pool. Each window views its own session, so you can have a different one live in each. If two windows point at the same session, the focused window shows it live and the other shows a muted frozen snapshot — focus a window and the live view follows.

Agent control CLI

The halo command drives and observes the running app over a unix socket (~/Library/Application Support/halo/control.sock, owner-only) — built so a script or an AI agent can run commands and read back what happened. Run bare halo while the app is open and it opens a new window in the running instance.

# open an isolated worktree session, split it, run tests, read the output
halo worktree fix/login
halo split -h
halo send-keys focused "npm test\n"
halo capture --scrollback

Lua scripting

Halo embeds Lua 5.4. On launch (and on halo reload) it runs ~/.config/halo/init.lua with a global halo table. A script error is shown as a toast — it never crashes the app. Everything the control CLI can do is reachable from Lua, plus UI you can't get from the CLI: floating panels, pickers, prompts, a statusline, and live config.

-- ~/.config/halo/init.lua

-- toast when a command/agent turn finishes in any session
halo.on("command-finished", function(id) halo.notify("done: "..id) end)

-- ctrl+g → git status in the focused pane
halo.bind("ctrl+g", function() halo.send("git status\n") end)

-- a command runnable with `halo run tests` or from a keybind
halo.command("tests", function()
  local p = halo.active()
  halo.notify("running tests in "..p.cwd)
  halo.send("npm test\n")
end)

The halo API

Register handlers (these persist across the script):

APIWhat it does
halo.command(name, fn)Register a command, runnable via halo run <name> or from a keybind.
halo.bind(chord, fn)Bind a key chord ("cmd+shift+p", "ctrl+g", …) to a function.
halo.on(event, fn)Run fn on an event; the handler gets the relevant paneID.
halo.timer(seconds, fn)Call fn every seconds (repeating). Cleared on reload.
halo.plugin("owner/repo")Declare a plugin; cloned from GitHub on first launch. See Plugins.

Read & drive the terminal (see below):

halo.active()Focused session: {cwd, title, paneID} (or nil).
halo.state()Full tree: {projects = {{name, path, sessions = {…}}, …}}.
halo.capture([scrollback])Focused pane's text as a string; pass true to include scrollback.
halo.send(text)Type into the focused pane (include "\n" to run).
halo.cmd(verb, …)Run any CLI verb; returns its result table.
halo.split([h]) · halo.tab(a) · halo.select(p, s)Split (true = horizontal); tab = "new"/"next"/"prev"/"close"; switch session.
halo.open(path) · halo.zoom() · halo.browser([url]) · halo.focus([id])Open a session at a path; zoom the pane; open a browser pane; focus/cycle panes.

Build UI (see below):

halo.notify(msg)Toast banner in the active window.
halo.panel(lines, opts)Create/update a floating panel; returns its id. Lines can be labels, clickable buttons, or editable fields ({input=true, placeholder=, action=fn}).
halo.close(id)Remove a panel by id.
halo.prompt(msg [, default], fn)Text-input overlay; fn(text) on submit. Optional pre-filled default.
halo.confirm(msg, fn)Yes/No dialog; fn(true|false).
halo.pick(items, fn [, opts])Filterable list; fn(label). Items are strings or {label, desc}. Hugs content by default.
halo.pickmulti(items, fn [, opts])Multi-select (Tab marks); fn(table_of_labels).
halo.menu(items [, opts])Action list; each {text, desc, action=fn} runs its own action on select.
picker opts (optional): {width, height, maxrows, maxheight} — default hugs content + scrolls; height forces a fixed-tall panel, maxrows/maxheight set the scroll threshold.
halo.status(text)Set the sidebar-footer statusline text.
halo.set(key, value)Override a config key from Lua (Lua wins). See Config from Lua.

Reading & driving the terminal

halo.cmd(verb, ...) runs any control verb and returns its result as a table; the named helpers (capture, state, split, …) wrap the common ones. This is how a Lua command inspects output and reacts — the same surface an external agent gets, in-process.

-- watch a build; toast pass/fail by scanning the pane after each turn
halo.on("command-finished", function()
  local out = halo.capture()          -- visible text; capture(true) for scrollback
  if out:find("FAIL") then halo.notify("❌ tests failed")
  elseif out:find("PASS") then halo.notify("✅ green") end
end)

-- count sessions across the whole sidebar
local n = 0
for _, p in ipairs(halo.state().projects) do n = n + #p.sessions end

-- open a worktree, split, run, read back — via raw cmd()
halo.cmd("worktree", "fix/login")
halo.split(true)
halo.send("npm test\n")

Custom UI: panels, prompts, pickers, statusline

A panel is a floating, titled box pinned to a corner. lines is a list where each entry is one of: a plain string or {text=, color="#hex"} — a label; {text=, color=, click=function() … end} — a button; or {input=true, placeholder=, action=function(text) … end} — an editable field whose action fires on Enter. Pass an existing id in opts to update a panel in place (e.g. from a timer) instead of stacking a new one.

panel opts—
titleHeader text.
corner"topright" (default), "topleft", "bottomright", "bottomleft".
bgBackground hex, e.g. "#1a1b26".
widthFixed width in points.
window"active" (default) — lives in the focused window and follows focus; "all" — shows in every window, including new ones.
idPass a returned id to update that panel instead of creating one.
 
-- a live clock that updates in place, in whichever window has focus
local clock
halo.timer(1, function()
  clock = halo.panel({
    { text = os.date("%H:%M:%S"), color = "#e0af68" },
  }, { title = "clock", corner = "topright", window = "active", id = clock })
end)

-- a button menu shown in EVERY window (click runs the function)
halo.panel({
  { text = "▶ split",   color = "#9ece6a", click = function() halo.split() end },
  { text = "＋ new tab", color = "#7aa2f7", click = function() halo.tab("new") end },
}, { title = "actions", corner = "bottomright", bg = "#16161e", window = "all" })

-- prompt for text, then run it; or pick from a list
halo.bind("cmd+shift+p", function()
  halo.prompt("Run:", function(cmd) if cmd ~= "" then halo.send(cmd.."\n") end end)
end)
halo.pick({ "git status", "npm test", "clear" }, function(c) halo.send(c.."\n") end)

-- a command palette: each item runs its own action
halo.menu({
  { text = "split",   desc = "vertical", action = function() halo.split() end },
  { text = "new tab", action = function() halo.tab("new") end },
})

halo.status("⏱ "..os.date("%H:%M"))   -- sidebar-footer statusline

Panels pass clicks through to the terminal underneath except on buttons, so a corner HUD never gets in your way. Non-active-window copies of an "active" panel simply disappear until that window is focused again.

Events & timers

halo.on(event, fn) subscribes; halo.timer(seconds, fn) polls. Handlers receive the relevant paneID where it applies.

session-openedA session/tab was created (arg: paneID).
session-closedThe user closed a session (arg: paneID).
session-exitedA shell exited on its own, vs. being closed (arg: paneID).
focus-changedThe active session changed (arg: paneID).
command-finishedA foreground command/agent turn returned to the shell (arg: paneID).
dir-changedThe focused pane's working directory changed (arg: paneID).
pane-outputRaw output bytes from any live pane: fn(paneID, chunk). Best-effort/coalesced; persist mode only. Use chunk:find(s, 1, true).
config-reloadedinit.lua (re)loaded — fires once after load and on every halo reload.

Plugins are sandboxed: a callback that errors repeatedly auto-disables its plugin (halo plugins enable <name> to restore), and an infinite loop in a callback is aborted rather than freezing the UI. Scrollback is persisted to disk, so a pane's history survives a daemon restart.

Config from Lua

halo.set(key, value) sets a config key from your script. Short chrome names (accent, surface, font-family, sidebar-width, font-size, divider-width) get the halo- prefix; any other key is treated as a raw Ghostty key (e.g. background) and reaches libghostty. Lua wins over the config file and the Settings UI — a key set here shows as locked with a “set by init.lua” badge in Settings.

-- a dark theme as a script: chrome + terminal share one background
halo.set("background", "#0a0b0f")   -- ghostty terminal bg
halo.set("surface",    "#0a0b0f")   -- halo chrome → halo-surface
halo.set("accent",     "#ff3366")   -- → halo-accent

Plugins

A plugin is just Lua using the same halo API. Declare one in init.lua and Halo clones it into ~/.config/halo/plugins/<name>/ on first launch; or drop a folder there yourself — no declaration needed. Each plugin's init.lua (or plugin/init.lua) runs at startup and registers its own commands, events, keybinds, and UI.

-- ~/.config/halo/init.lua
halo.plugin("you/halo-statusbar")                       -- latest from github.com/you/halo-statusbar
halo.plugin("you/halo-fzf", { ref = "v1.2.0", priority = 10 })   -- pinned + loads first

-- ~/.config/halo/plugins/git-hud/init.lua  (a complete drop-in plugin)
local id
halo.timer(2, function()
  local a = halo.active(); if not a then return end
  id = halo.panel({
    { text = a.cwd, color = "#7dcfff" },
    { text = "⟳ fetch", color = "#9ece6a", click = function() halo.send("git fetch\n") end },
  }, { title = "git", corner = "topleft", window = "all", id = id })
end)
Manage—
halo pluginsList installed plugins; disabled ones are marked.
halo plugins syncUpdate each plugin and reload — pinned plugins fetch + checkout their ref, the rest fast-forward.
halo plugins enable|disable <name>Toggle a plugin (persisted) and reload. Also in Settings ▸ Plugins.

Disabling a plugin re-runs config cleanly, so a theme plugin's colors revert the moment you turn it off — no relaunch.

Pinning, manifest & lockfile

Pass a table to halo.plugin to pin a ref (tag, branch, or commit) and/or set a load priority (higher loads first; ties break by name). A plugin can also self-describe with an optional manifest.lua — useful for drop-ins, since they have no declaration. Declared options win over the manifest.

-- a plugin's manifest.lua
return { version = "1.2.0", priority = 10 }

Every load writes ~/.config/halo/plugins.lock — the resolved commit, ref, and version of each installed plugin — so an install is reproducible and you can see exactly what's running. halo plugins shows it inline:

halo plugins
→ halo-fzf      v1.2.0  @v1.2.0  (9283de5)
→ halo-statusbar         (a1b2c3d)
→ git-hud       -- drop-in, not a git repo

halo.plugin also accepts a full URL (https://…, git@…, file://…) or an absolute path, not just owner/repo — handy for private repos and local plugin development.
